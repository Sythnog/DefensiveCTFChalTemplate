How do I solve a defensive CTF challenge?

To solve a defensive CTF challenge you need to find and solve a security vulnerability in a service. Once you feel that you have solved the vulnerability, then you can ask the judge (this service) to validate that the vulnerability has been solved. Furthermore, it will perform checks to validate that the "normal" functionality still works.

HINT: It might be useful to run the judge and see which requests are sent, as it will help you find the vulnerability to solve. Furthermore, the following is a rough "game-plan" of how you might approach the challenge:

  • Connect to the vulnerable service via: ssh root@{{ .challWebLocation }}
  • The password for SSH is {{ .challSSHPass }}
  • You can find the files for the vulnerable service in: {{ .challFileLoc }}
  • Now you need to find and solve the vulnerability in the service.
  • When the vulnerability is solved, restart the service with: {{ .challRestartServiceCMD }}
  • Go to the main page of this service and start the judge.
  • The judge will grant you the flag, if the vulnerability has been solved and "normal" functionality still works.

Which tools do I have available?

  • We have pre-installed a handful of useful tools on the vulnerable machine:
  • Text-editors: vim, nano, emacs
  • Send requsts to download scripts/fixes, etc.: curl, wget
  • Network analysis: tcpdump
  • Furthermore, a handfull of technology-specific tools are on the vulnerable machine.
